## 1. 概要・目的

### 1.1 背景
福利厚生サービス「リアライズクラブ（RC）」での企業導入を入口に、**完全予約制でマイホーム提案へつなぐ B2B2C 導線**（賃貸窓口 → マイホーム）を支える営業支援ツール。「人生相談から住居費設計へ」視点を移す、トップクローザー育成型の営業プロセスをシステム化する。

### 1.2 目的
- 反響〜契約までの商談プロセス（反響・予約 → 人生相談 → プレゼン → 交渉 → 契約）を1ツールで可視化・管理
- **法人（BtoB）・個人（BtoC）どちらの顧客も**管理できるCRM
- 面談音声を**AIで自動議事録化**し、入力作業を削減
- 飛込〜2回目の**アンケート（幸せ意識度チェック／ライフメイクカルテ）**を記録
- クローザー育成（標準カリキュラム＋習得度管理）
- **チームで使える**（ユーザー別ログイン＋登録コードによる新規登録）

### 1.3 設計方針
- **自己完結 ＋ 段階拡張**：APIキー無し（デモモード）でも全機能が動作し、キーを入れると本物のAIに切替
- **1コンテナ完結**：FastAPI 1プロセス ＋ SQLite。ローカルでもVPSでも同一コードで動作
- **データはユーザー管理下**：VPSの永続ディスクにSQLiteで保存（消えない）

## 2. システム全体像

### 2.1 本番構成（Xserver VPS）
```
[利用者のブラウザ / スマホ]
        │ HTTPS (https://sales.lifemakepartners.net)
        ▼
[DNS: 別Xserverで管理] sales.lifemakepartners.net → 162.43.38.50 (Aレコード)
        ▼
[Xserver VPS  162.43.38.50 / Ubuntu 22.04]
  ├ nginx（リバースプロキシ / Let's Encrypt HTTPS）
  │    ・server_name sales.lifemakepartners.net → 127.0.0.1:8123
  │    ・既存サイト(lmp-elearning.com 等)とサブドメインで共存（既存に不干渉）
  ├ systemd サービス "lms"（24時間常駐・自動再起動）
  │    └ uvicorn main:app  (127.0.0.1:8123)
  │         ├ FastAPI（業務API / 認証 / AI議事録）
  │         ├ SQLite  /opt/life-make-sales/sales_tool.db（永続）
  │         └ 常駐ワーカースレッド（AI議事録のDBキュー処理）
  └ 外部AI（任意・キー設定時）: OpenAI Whisper / AssemblyAI / Anthropic Claude
```

### 2.2 技術スタック
| レイヤ | 採用技術 |
|--------|----------|
| バックエンド | Python / FastAPI / Uvicorn |
| データベース | SQLite（標準ライブラリ・追加DB不要） |
| フロントエンド | バニラJavaScript SPA（ビルド不要）、Noto Sans JP |
| 認証 | 自前（users/sessions・pbkdf2-sha256・Cookieセッション） |
| 文字起こし | OpenAI Whisper API ／ ローカル faster-whisper（任意） |
| 話者分離 | AssemblyAI（任意） |
| 議事録・提案トーク生成 | Anthropic Claude |
| 配置 | Xserver VPS（nginx＋systemd＋certbot）／ GitHub でコード同期 |

### 2.3 配布・移植
- GitHub 公開リポジトリ `nrealize6078-hue/life-make-sales` でコード管理
- ローカル：`起動.bat`／`setup.sh`　クラウド：Docker/Render 定義も同梱
- 本番：VPSで `git pull` → `systemctl restart lms` で更新（データは保持）

## 4. 画面設計（SPA）

- **サイドバー**：3セクション（ホーム／営業活動／管理・育成）でビュー切替。下部にユーザー名・PW変更・ユーザー管理・ログアウト
- **トップバー**：全ページ上部中央に会社ブランド「LIFE MAKE PARTNERS ／ ロゴ ／ ライフメイクパートナーズ」を横1行表示。左に画面タイトル、右に画面別アクション
- **ログイン画面**：ユーザー名＋パスワード。登録コード設定時は「新規登録」を表示
- **モーダル**：追加・編集・詳細を共通モーダルで処理。**トースト**通知
- **レスポンシブ**：狭幅でサイドバーをドロワー化（ハンバーガー）、テーブルをカード化。ブランド表記は狭幅で非表示

## 5. データベース設計（SQLite）

全テーブルが顧客(companies)・商談(deals)を軸に連動。外部キーは原則 `ON DELETE SET NULL`（進捗・セッション・育成進捗は CASCADE）。起動時に不足カラムを自動追加する簡易マイグレーションを実装。

| テーブル | 役割・主なカラム |
|----------|------------------|
| `companies` | 顧客。name, **customer_type(btob/btoc)**, industry, address, phone, **email**, website, notes, hubspot_id |
| `contacts` | 担当者（法人）。company_id, name, title, email, phone |
| `deals` | 商談。title, stage, amount, probability, expected_close, owner, presentation(4章JSON), hubspot_id |
| `tasks` | タスク。title, source_text, due_date, priority, status, company_id, deal_id |
| `meetings` | 面談＋AI議事録。scheduled_at, location, meeting_type, status, agenda, minutes, **attendees(同席者)**, audio_filename, transcript, summary, minutes_md, decisions/next_actions/tags(JSON), ai_status, auto_generate |
| `hearing_sheets` | 人生相談カルテ。6大項目＋raw_voice_text＋talk_points(JSON) |
| `happiness_checks` | 幸せ意識度チェック。answers(JSON), no_count, memo |
| `lifemake_kartes` | ライフメイクカルテ。data(全項目JSON) |
| `progress_logs` | **顧客の日付ごとの進捗状況**。company_id, log_date, status, note |
| `training_modules` / `training_progress` | 育成カリキュラム／習得進捗 |
| `users` | ユーザー。username(uniq), display_name, password_hash(pbkdf2), role(admin/member), active |
| `sessions` | ログインセッション（トークン失効対応）。token, user_id, expires_at |
| `app_settings` | アプリ設定 key-value（**register_code=登録コード** 等） |

## 6. ドメインロジック

- **商談ステージ**：反響・予約 → 人生相談 → プレゼン → 交渉 → 契約 ／ 失注（契約=確度100%、失注=0%自動）
- **4章プレゼン**：①家は買うべき ②今買うべき ③ここを買うべき ④うちから買うべき（商談ごとにJSONで進捗保持）
- **幸せ意識度チェック**：7問のNO数を集計し、NO≥3で「マイホーム見込み」
- **顧客区分**：btob（法人：企業名・業種・担当者）／btoc（個人：氏名・電話・メール・住所中心）
- **育成カリキュラム**：トップクローザー研修準拠（基礎/人生相談/プレゼン/交渉/契約/集客/ツール活用）

## 8. AI議事録パイプライン（DBキュー方式）

```
[POST /api/minutes/upload]
  1. 音声保存＋meetings行を ai_status='queued' で作成（即応答）
        ↓（常駐ワーカースレッドが queued を1件ずつ処理）
  2. processing → 文字起こし（話者分離/ローカルWhisper/OpenAI Whisperを設定で選択）
        ↓ transcribed
  3. summarizing → Claudeで議事録一式生成（要約/議事録MD/決定事項/次アクション/タグ）
        ↓ summarized（完了）／失敗時 error
```
- Redis等の外部インフラ不要・1コンテナ完結。SQLite書込競合対策に busy_timeout 設定
- 再起動時は中断ジョブを queued に戻して自動再処理（耐障害性）

## 9. 認証・新規登録設計

| 項目 | 内容 |
|------|------|
| ログイン | ユーザー名＋パスワード。パスワードは pbkdf2-sha256（salt付） |
| セッション | Cookie `lms_session`（httponly/samesite=lax/7日）。sessionsテーブルで**失効可能** |
| ロール | admin（全機能・ユーザー管理）／ member（業務機能のみ） |
| 保護 | ミドルウェアで `/api/*` を保護（`/api/auth/*`・ヘルスは除外） |
| **新規登録** | 登録コード（合言葉）方式。管理者がユーザー管理で**登録コードを設定**すると、ログイン画面に「新規登録」が出現。**コードを知る人だけ**メンバー登録＆自動ログイン。空欄で登録停止 |
| 初期admin | 初回起動時 `APP_PASSWORD` から自動作成。`vpsfix.sh` 等で再設定可能 |

## 10. デプロイ・運用（Xserver VPS）

### 10.1 構成
- VPS：Xserver VPS（162.43.38.50 / Ubuntu 22.04 / 2GB）
- Web：nginx リバースプロキシ（`/etc/nginx/conf.d/lms.conf`、server_name=sales.lifemakepartners.net、既存サイトと共存）
- 常駐：systemd サービス `lms`（uvicorn 127.0.0.1:8123・自動再起動・起動時開始）
- HTTPS：Let's Encrypt（certbot・自動更新）
- データ：`/opt/life-make-sales/sales_tool.db`（永続）＋アップロード音声

### 10.2 設置・更新スクリプト（GitHub同梱）
- `vps_setup.sh`：nginx共存で新規設置（依存導入→clone→systemd→nginx）
- `vps_update.sh` / 手順 `git pull`＋`systemctl restart lms`：更新（データ保持）
- `vpsfix.sh`：更新＋adminパスワード再設定
- `vps_https.sh`：certbotでHTTPS化

### 10.3 更新手順（機能追加時）
```
cd /opt/life-make-sales      # コンソールでは cd /opt/life* でも可
git pull
systemctl restart lms         # バックエンド変更時のみ（画面だけなら git pull のみ）
```

### 10.4 主要な認証情報（※取扱注意・社外秘）
| 対象 | ID | 補足 |
|------|----|----|
| VPSコンソール(SSH/VNC) | `root` | root パスワード（別途厳重管理） |
| アプリ管理者 | `admin` | 管理者パスワード（別途厳重管理・PW変更可） |
| 新規メンバー | 各自 | 登録コードで自己登録 |

## 12. 既知の課題・今後の拡張

- **AI本番利用**：VPSの `.env` で `DEMO_MODE=false` ＋各APIキー設定で本物のAIが有効
- **データ基盤**：規模拡大時に SQLite → PostgreSQL（`database.py` 差し替え）
- **バックアップ**：`/opt/life-make-sales/sales_tool.db` を定期コピー推奨
- **ジョブ基盤**：さらなる並列化時は Redis＋ワーカー分離
- **権限**：現状 admin/member の2段階。より細かい権限が必要なら拡張
